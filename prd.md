# Typo Trap PRD

## 1. 프로젝트 개요

Typo Trap은 제한 시간 안에 여러 단어 중 오타가 없는 올바른 한글 단어를 찾아 클릭하는 미니게임이다.

플레이어는 짧은 시간 안에 단어를 빠르게 관찰하고, 비슷하게 생긴 오타 단어들 사이에서 정답을 선택해야 한다.

이 프로젝트는 Pixi.js를 사용하여 개발하는 독립형 웹 미니게임이다.

## 2. 프로젝트 목표

* Pixi.js 기반의 독립 미니게임을 완성한다.
* 한글 단어 기반의 간단하고 직관적인 게임을 만든다.
* 초등학생도 이해할 수 있는 쉬운 단어를 사용한다.
* 제한 시간 안에 정답을 찾는 긴장감 있는 플레이 경험을 제공한다.
* 복잡한 디자인보다 게임 규칙, 상태 관리, 화면 전환, 플레이 완성도를 우선한다.

## 3. 핵심 게임 콘셉트

플레이어는 화면에 표시된 여러 단어 중에서 **오타가 없는 정답 단어**를 찾아야 한다.

예시:

```txt
정답: 사자

보기:
사자
샤자
사쟈
샂자
```

플레이어가 `사자`를 클릭하면 성공한다.
오타 단어를 클릭하면 즉시 게임 오버가 된다.

## 4. 타겟 사용자

* 짧고 가벼운 웹 미니게임을 좋아하는 사용자
* 반응 속도 게임을 좋아하는 사용자
* 한글 단어를 빠르게 구분하는 게임을 즐기고 싶은 사용자
* 간단하지만 집중력이 필요한 캐주얼 게임을 원하는 사용자

## 5. 게임 규칙

### 5.1 기본 규칙

* 게임은 총 5단계로 구성한다.
* 각 단계의 제한 시간은 3초다.
* 화면에는 정답 단어 1개와 오타 단어 여러 개가 표시된다.
* 플레이어는 제한 시간 안에 정답 단어를 클릭해야 한다.
* 오답을 클릭하면 즉시 게임 오버가 된다.
* 제한 시간이 끝나도 게임 오버가 된다.
* 5단계를 모두 클리어하면 성공 화면을 보여준다.
* 초기 버전에서는 점수 시스템을 제공하지 않는다.

### 5.2 단어 규칙

* 단어는 한글만 사용한다.
* 초등학생도 알 수 있는 쉬운 단어를 우선 사용한다.
* 정답 단어는 2~4글자 단어를 우선 사용한다.
* 오타는 비슷하게 생긴 글자로만 만든다.
* 한 글자 빠짐, 글자 추가, 순서 바뀜 오타는 초기 버전에서 사용하지 않는다.

### 5.3 오타 예시

```txt
사자 → 샤자
바다 → 바따
토끼 → 도끼
나무 → 나므
고양이 → 고양니
```

오타는 정답 단어와 비슷해 보여야 하지만, 정답과 정확히 일치하지 않아야 한다.

## 6. 화면 구성

### 6.1 시작 화면

목적:

* 게임 제목과 간단한 규칙을 보여준다.
* 플레이어가 직접 게임을 시작할 수 있게 한다.

포함 요소:

* 게임 제목: Typo Trap
* 간단한 설명
* 시작 버튼

예시 문구:

```txt
3초 안에 오타 없는 단어를 찾아라!
틀리면 바로 게임 오버!
```

### 6.2 게임 화면

```
기존의 남은 시간 텍스트 표시 대신, 남은 시간을 Progress Bar로 보여준다.

게임 화면 포함 요소:

현재 단계 표시
남은 시간을 나타내는 Progress Bar
단어 보기 목록
클릭 가능한 단어 버튼 또는 텍스트

예시:

Stage 1 / 5

[████████████████████] 3초

사자    샤자    사쟈    샂자

Progress Bar는 스테이지 시작 시 가득 찬 상태에서 시작하고, 3초 동안 점점 줄어든다.
Progress Bar가 모두 줄어들면 시간 초과로 게임 오버가 된다.
```

### 6.3 정답 화면

목적:

* 정답을 맞혔다는 피드백을 보여준다.
* 자동으로 다음 단계로 넘어가지 않고, 사용자가 버튼을 눌러 다음 단계로 이동한다.

포함 요소:

* 정답 피드백
* 다음 단계 버튼

예시 문구:

```txt
정답!
다음 단계로 가볼까요?
```

### 6.4 게임 오버 화면

목적:

* 오답 클릭 또는 시간 초과로 게임이 끝났음을 알려준다.
* 다시 시작할 수 있게 한다.

포함 요소:

* 게임 오버 문구
* 실패 이유
* 다시 시작 버튼

실패 이유:

* 오답 클릭
* 시간 초과

예시 문구:

```txt
Game Over
오타 단어를 클릭했어요!
```

또는

```txt
Game Over
시간이 다 됐어요!
```

### 6.5 클리어 화면

목적:

* 5단계를 모두 통과했음을 알려준다.
* 다시 시작할 수 있게 한다.

포함 요소:

* 클리어 문구
* 다시 시작 버튼

예시 문구:

```txt
Clear!
모든 오타를 피해냈어요!
```

## 7. 게임 상태

게임은 아래 상태를 가진다.

```txt
ready       시작 전
playing     게임 진행 중
correct     정답 선택 후 다음 단계 대기
gameOver    실패
clear       전체 클리어
```

## 8. 기능 요구사항

### 8.1 필수 기능

* 시작 화면에서 게임 시작 가능
* 단계별 단어 데이터 표시
* 제한 시간 3초 카운트다운
* 정답 클릭 시 정답 상태로 전환
* 오답 클릭 시 게임 오버
* 시간 초과 시 게임 오버
* 다음 단계 버튼 클릭 시 다음 단계 시작
* 5단계 완료 시 클리어 화면 표시
* 다시 시작 기능 제공
* Pixi.js 리소스 정리 기능 제공

### 8.2 제외 기능

초기 버전에서는 아래 기능을 만들지 않는다.

* 로그인
* 랭킹
* 점수 저장
* 난이도 선택
* 사운드 설정
* 모바일 최적화 고도화
* 애니메이션 고도화
* 디자인 시스템
* 외부 서버 연동

## 9. 기술 요구사항

* Vite
* TypeScript
* Pixi.js
* ESLint
* Prettier
* 독립 실행 가능한 웹 미니게임 프로젝트

## 10. 비기능 요구사항

* 게임 로직과 렌더링 로직을 최대한 분리한다.
* 단어 데이터는 별도 파일로 관리한다.
* 게임 상태 전환은 명확하게 관리한다.
* Pixi Application은 destroy 시 안전하게 정리한다.
* 게임은 브라우저에서 단독으로 실행 가능해야 한다.
* 초기 버전은 단순하고 안정적인 동작을 우선한다.

## 11. 성공 기준

초기 버전의 성공 기준은 다음과 같다.

* `pnpm dev` 실행 시 게임이 브라우저에서 열린다.
* 시작 버튼을 누르면 게임이 시작된다.
* 각 단계마다 3초 제한 시간이 동작한다.
* 정답을 클릭하면 다음 단계로 이동할 수 있다.
* 오답을 클릭하면 즉시 게임 오버가 된다.
* 시간 초과 시 게임 오버가 된다.
* 5단계를 클리어하면 클리어 화면이 나온다.
* 다시 시작 버튼으로 게임을 처음부터 다시 할 수 있다.
* 콘솔에 치명적인 에러가 없어야 한다.

## 12. 향후 확장 아이디어

초기 버전 완성 후 아래 기능을 고려할 수 있다.

* 난이도 추가
* 더 많은 단어 데이터 추가
* 랜덤 스테이지 생성
* 콤보 시스템
* 점수 시스템
* 로컬 최고 기록 저장
* 모바일 터치 UX 개선
* 효과음과 배경음 추가
* 간단한 애니메이션 연출 추가
